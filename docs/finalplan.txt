1. AWS ARCHITECTURE

Frontend Hosting:
- Service: AWS Amplify
- Why:
  - Automatic CI/CD from Git
  - Built-in SSL certificates
  - Custom domain support
  - CDN included
  - Easy rollbacks
- Cost: ~$1-5/month

Backend API:
- Service: AWS App Runner (Express.js) OR AWS Lambda + API Gateway
- Recommendation: Start with App Runner (easier), migrate to Lambda later
- Why:
  - Deploy existing Express code with minimal changes
  - Auto-scaling
  - Managed service
- Cost: ~$5-25/month (App Runner) or ~$0-5/month (Lambda)

Database:
- Service: Amazon RDS PostgreSQL (or Aurora Serverless v2)
- Recommendation: RDS PostgreSQL db.t3.micro (Free Tier eligible)
- Storage: 20GB SSD
- Backup: Automated daily backups
- Cost: $15-25/month (or Free Tier for 12 months)

Email Service:
- Service: Amazon SES
- Features:
  - Lead notification emails to you
  - Auto-reply to customers
- Cost: $0.10 per 1,000 emails (basically free)

Domain & SSL:
- Service: AWS Route 53 + AWS Certificate Manager (ACM)
- Features:
  - DNS management
  - Free SSL certificates
  - Health checks
- Cost: $0.50/month per hosted zone + $0.40 per million queries

2. DATABASE SCHEMA

-- Leads Table
CREATE TABLE leads (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name VARCHAR(255) NOT NULL,
  email VARCHAR(255) NOT NULL,
  phone VARCHAR(50),
  company VARCHAR(255),
  service_interest VARCHAR(100), -- which AI service they're interested in
  message TEXT,
  source_page VARCHAR(100), -- which page they submitted from
  created_at TIMESTAMP DEFAULT NOW(),
  status VARCHAR(50) DEFAULT 'new', -- new, contacted, qualified, closed
  notes TEXT -- admin notes
);

-- Admin Users Table (simple auth)
CREATE TABLE admin_users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  username VARCHAR(100) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Indexes for performance
CREATE INDEX idx_leads_created_at ON leads(created_at DESC);
CREATE INDEX idx_leads_status ON leads(status);
CREATE INDEX idx_leads_email ON leads(email);

4. DOMAIN CONFIGURATION

Steps to Connect Your Domain:

Option A: Domain Already Registered Elsewhere (GoDaddy, Namecheap, etc.)
1. Keep domain at current registrar
2. Point nameservers to AWS Route 53
3. Route 53 manages DNS
4. Connect to Amplify

Option B: Transfer Domain to AWS Route 53
1. Transfer domain to AWS
2. AWS manages everything
3. Slightly easier but takes 5-7 days

SSL Certificate:
- AWS Certificate Manager (ACM) provides FREE SSL
- Auto-renewal
- Managed by AWS

DNS Records:
yourdomain.com        → AWS Amplify (A record)
www.yourdomain.com    → AWS Amplify (CNAME)
api.yourdomain.com    → App Runner API (CNAME) [optional]
